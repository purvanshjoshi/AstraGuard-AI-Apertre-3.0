## ğŸ“‹ Badge Assignment Process

### Automatic Assignment

Tier badges are automatically assigned based on:
- **PR Count**: Number of merged pull requests
- **Quality Score**: Code review feedback and CI/CD results
- **Consistency**: Regular contribution over time

```python
# Pseudocode for automatic badge assignment
def assign_tier_badge(contributor):
    pr_count = get_merged_pr_count(contributor)
    
    if pr_count >= 50:
        return "Legend"
    elif pr_count >= 20:
        return "Core"
    elif pr_count >= 5:
        return "Regular"
    elif pr_count >= 2:
        return "Active"
    elif pr_count >= 1:
        return "New"
    
    return None
```

### Manual Assignment

Specialty and award badges are manually assigned by maintainers based on:
- **Contribution Type**: Specific area of expertise
- **Community Impact**: Helpfulness and mentorship
- **Quality of Work**: Excellence in specific domain
- **Nominations**: Peer recognition

## ğŸ¨ Custom Badge Requests

Contributors can request custom badges for:
- **Unique Contributions**: Special projects or research
- **External Recognition**: Publications, talks, articles
- **Long-term Dedication**: Years of consistent contribution
- **Special Achievements**: Exceptional milestones

**How to Request:**
1. Open an issue with the `badge-request` label
2. Describe your contribution or achievement
3. Provide supporting evidence (links, metrics, etc.)
4. Wait for maintainer review
